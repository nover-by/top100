# Unbiased Top 100

A Bayesian correction of [DR's Top 100](https://www.dr.dk/feature/top100) for alphabetical voting bias.

## The Problem

DR's Top 100 was voted by listeners who each picked up to 5 songs from a long list sorted **alphabetically by artist name**. This creates a systematic "primacy effect": artists whose names start with letters early in the alphabet receive more votes simply because they appear at the top of the list — not because their songs are better.

## The Model

We fit a Poisson regression in PyMC (non-centered hierarchical parameterisation):

```
votes_i  ~ Poisson(μ_i)
log(μ_i) = μ_global + α_i + β × pos_i

α_i = alpha_offset_i × σ    # non-centered; latent song quality
β   ~ Normal(0, 0.5)         # alphabetical bias coefficient
σ   ~ HalfNormal(1)
```

- `pos_i` is the alphabetical position of the artist's first letter (A=1 … Å=29, Danish), standardised to zero mean and unit variance.
- `votes_i` is a proxy derived from rank: `votes = 101 − rank` (rank 1 → 100 votes, rank 100 → 1 vote).
- After sampling, songs are ranked by **posterior mean of `α_offset_i × σ`** — the estimated song quality free of alphabetical influence.

A **negative β** confirms the alphabetical bias: artists near the start of the alphabet received artificially inflated votes.

## Setup & Running

### 1. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate   # macOS / Linux
# .venv\Scripts\activate    # Windows
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Run the model

```bash
python model.py
```

This will:
- Parse `top100.txt`
- Sample 4 NUTS chains (≈2–5 minutes on a modern laptop)
- Print the β posterior summary and unbiased top 10
- Write `docs/results.json`

### 4. Preview the website locally

```bash
python -m http.server 8000 --directory docs
```

Open <http://localhost:8000> in your browser.

## Deploying to GitHub Pages

1. Push the repository (including `docs/results.json`) to GitHub.
2. Go to **Settings → Pages**.
3. Set **Source** to `Deploy from a branch`, branch `main`, folder `/docs`.
4. Save — your site will be live at `https://<username>.github.io/<repo>/`.

## Interpreting the Results

| Field | Meaning |
|---|---|
| `rank_biased` | Original DR ranking |
| `rank_unbiased` | Model-estimated true ranking |
| `rank_delta` | `rank_biased − rank_unbiased`; positive = song moved up after debiasing |
| `bias_effect` | `β_raw × alpha_pos` — log-vote boost/penalty from the alphabet |
| `posterior_quality` | Posterior mean of `α_offset × σ` — the latent song quality score |

## Notes

- **Vote proxy**: Using `101 − rank` as a linear vote proxy is a simplification. The absolute scale is arbitrary; only the relative ordering by `posterior_quality` matters.
- **ESS warning**: PyMC may report low ESS for `sigma` — normal when each song has only one observation. The rankings are robust to this.
- **Re-running**: Each run produces slightly different results due to MCMC stochasticity (fixed with `random_seed=42` for reproducibility).
